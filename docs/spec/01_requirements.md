# 要件定義

## 機能要件

### FR-01 ファイルアップロード
- Web UIからファイルをアップロードできる
- 対応形式: 動画（mp4/mkv/avi/mov）、音声（mp3/wav/m4a/ogg/flac）、テキスト（txt/md）
- ファイルサイズ上限: 2GB

### FR-02 音声文字起こし【必須】
- 動画・音声ファイルから音声を抽出し、テキストに変換する
- 日本語・英語に対応する
- タイムスタンプ付きで出力する
- 使用モデル: faster-whisper（small モデル）
- **常に実行される（スキップ不可）**

### FR-03 スライド・画面資料の抽出【任意】
- 動画からシーン変化（スライド切り替え）を検出する
- 変化点のフレームを画像として保存する
- 各フレームからOCRでテキストを抽出する
- **`enable_ocr=true` かつ動画ファイルの場合のみ実行される**

### FR-04 要約生成【任意】
- 文字起こしテキストとスライドテキストを入力として要約を生成する
- 使用モデル: Ollama（qwen2.5:3b）
- 出力形式: Markdown
- **`enable_minutes=true` の場合のみ実行される**

### FR-05 議事録生成【任意】
- 要約をもとに構造化された議事録を生成する
- 含む項目: タイトル・日時・概要・主要トピック・決定事項・TODO
- 出力形式: Markdown
- **`enable_minutes=true` の場合のみ実行される（FR-04 と同時実行）**

### FR-06 処理結果のダウンロード
- 各出力ファイルをWeb UIからダウンロードできる
- 一括ZIPダウンロードに対応する

### FR-07 処理状況の表示
- 処理の進捗をリアルタイムで表示する
- エラー発生時はメッセージを表示する

## 非機能要件

### NFR-01 セキュリティ・プライバシー
- ファイルデータは外部ネットワークに送信しない
- 処理はすべてローカルで完結する

### NFR-02 環境
- OS: Windows 11
- Python: 3.10 以上
- Docker: 対応（GPU不要で動作）
- GPU: NVIDIA CUDAがあれば高速化（なくても動作）

### NFR-03 可用性
- Docker Composeで起動・停止できる
- 初回以外はオフライン環境で動作する

### NFR-04 保守性
- 仕様書駆動で開発する
- 各モジュールは単一責任とする
- 設定値はyamlファイルで一元管理する
